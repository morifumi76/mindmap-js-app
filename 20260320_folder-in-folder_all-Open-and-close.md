# 3. 修正A：左サイドバーのフォルダ多階層化

## 3.1 現状

左サイドバーは **2階層構造**（フォルダ → マップページ）で管理されている。

フォルダの中にフォルダを作ることはできない。

## 3.2 変更内容

**Finderのリスト表示** をモデルにした、再帰的なフォルダ構造を実装する。

### 基本仕様

- フォルダの中に **フォルダとマップページの両方** を配置できる
- **階層の深さに制限なし**（再帰的構造）
- フォルダとマップページは **同じ階層に混在** できる（Finderと同じ）
- 各階層はインデントで深さを表現する
- フォルダの展開/折りたたみ（▼/▶）は現行通り維持

### UIイメージ

```
📂 マイマップ                    «
───────────────────────
[アルファベット順 ▼]
[＋ 新規作成]

▼ 📁 メモ                    ⋯
    ▼ 📁 old                 ⋯    ← サブフォルダ
        メモ1              ⋯    ← マップページ
        メモ2              ⋯
    メモ3                    ⋯    ← フォルダ直下のマップ
    ▶ 📁 アイデア             ⋯    ← 折りたたみ状態のサブフォルダ
▼ 📁 仕事                    ⋯
    ▼ 📁 進行中              ⋯
        ▼ 📁 TEMP社          ⋯    ← さらに深い階層もOK
            案件A           ⋯
        案件B               ⋯
    ▶ 📁 アーカイブ           ⋯

[📁 フォルダを作成]
```

### インデント仕様

- 階層1つあたり **20px** の左パディングを追加（Finderに近い値）
- マップページもフォルダも、同じ階層なら同じインデント量

### フォルダの ⋯メニュー（変更）

現行のフォルダメニューに **「📁 フォルダを追加」** を追加する。

| 操作 | 動作 |
| --- | --- |
| ✏️ 名前変更 | 現行通り |
| 📄 ページを追加 | 現行通り（このフォルダ直下にマップを作成） |
| 📁 フォルダを追加 | **NEW：** このフォルダの中にサブフォルダを作成。デフォルト名は「新しいフォルダ」 |
| 🗑 削除 | 現行通り（中身のマップ・サブフォルダも含めて削除。確認ダイアログあり） |

### ドラッグ&ドロップ

- マップページをドラッグして **任意のフォルダ（どの階層でも）** に移動可能
- **フォルダ自体** もドラッグして、別のフォルダの中や別の位置に移動可能
- フォルダをフォルダの中にドロップ → そのフォルダのサブフォルダになる
- 自分自身の子孫フォルダにはドロップ不可（循環参照の防止）

### localStorageデータ構造の変更

現行の `mindmap-meta` にある `folderId` を、フォルダ自体も再帰的に持てるようにする。

**フォルダデータの構造案：**

```json
{
  "folders": [
    {
      "id": "folder-1",
      "name": "メモ",
      "parentFolderId": null,
      "order": 0
    },
    {
      "id": "folder-1-1",
      "name": "old",
      "parentFolderId": "folder-1",
      "order": 0
    },
    {
      "id": "folder-1-2",
      "name": "アイデア",
      "parentFolderId": "folder-1",
      "order": 1
    }
  ]
}
```

- 各フォルダが `parentFolderId` を持つことで、無限階層を表現
- `parentFolderId: null` はルート階層のフォルダ
- マップページの `folderId` は現行通り、所属するフォルダのIDを指す

### マイグレーション

- 既存の2階層データ（フォルダ + マップページ）は変更なしでそのまま動くようにする
- データ構造の変更は後方互換を保つ（既存の `folderId` はそのまま有効）

### 「未分類」フォルダの扱い

- 現行通り、フォルダに所属しないマップは「📁 未分類」に自動分類

# 4. 修正B：「すべて開く / すべて閉じる」ボタンの移設

## 4.1 現状

「すべて開く」ボタンは **右サイドバーのコントロールエリア**（下部）に配置されている。

「すべて閉じる」ボタンは存在しない（「すべて開く」のみ）。

## 4.2 変更内容

### 右サイドバーのコントロールエリア変更

- 右サイドバーのコントロールエリアから「すべて開く」ボタンを **削除** する
- コントロールエリアのレイアウトは現行の位置（右サイドバー下部）を維持
- 残る要素は上から順に：**🐣 ひよこモード トグル → 罫線トグル → コピーボタン** の3つ
- コピーボタンは、旧「すべて開く」ボタンがあった位置に配置（右寄せ）

**変更後のコントロールエリア：**

```
┌───────────────────────────┐
│  （上部：階層ツリー）        │
│  ─────────────────────── │
│  🐣 ひよこモード      [==●] │
│  罫線                [●==] │
│                      [コピー] │
└───────────────────────────┘
```

- 「コピー」ボタンと「すべて開く」ボタンが横並びだった現行デザインから、「すべて開く」を削除するだけ
- 「コピー」ボタンのデザイン・サイズは現行通り（幅100px、高さ32px）

### コントロールエリアの固定レイアウト（★重要）

**現状の問題：** 右サイドバーを横に広げると、ひよこモードのラベルとトグルスイッチの間、罫線のラベルとトグルスイッチの間が大きく開いてしまう。また、コピーボタンも横幅いっぱいに伸びてしまう。

**修正内容：** コントロールエリア内の全要素を、サイドバーの幅に関係なく **右寄せ・固定サイズ** にする。

- **トグル行（ひよこモード・罫線）：** ラベルとトグルスイッチを `flex` で右寄せにまとめる。サイドバーを広げても、ラベルとトグルの間隔は一定（広がらない）。具体的には、トグル行全体を `justify-content: flex-end` にするか、トグル行の内容を `max-width` で固定して右寄せにする
- **コピーボタン：** 幅は **100px固定**（`width: 100px; min-width: 100px; max-width: 100px;`）のまま、**右寄せ配置**（`margin-left: auto;`）にする。サイドバーがどれだけ広がってもボタンサイズは変わらない
- **コントロールエリア全体：** 右寄せ配置。サイドバー幅が変わっても、コントロール群は右端に固定されたまま動かない

**イメージ（サイドバーを広く開いた場合）：**

```
┌──────────────────────────────────────────────┐
│  （上部：階層ツリー）                            │
│  ────────────────────────────────────────── │
│                          🐣 ひよこモード [==●] │  ← 右寄せ固定
│                          罫線           [●==] │  ← 右寄せ固定
│                                       [コピー] │  ← 100px固定・右寄せ
└──────────────────────────────────────────────┘
```

### キャンバス上にアイコンボタンを追加

ズームコントロール（ `[ − ]` `[ 100% ]` `[ + ]` ）の 左**横** に、以下の2つのアイコンを追加する。

| アイコン | 機能 | 記号 |
| --- | --- | --- |
| `«` | **すべて閉じる** | 左向き二重シェブロン（U+00AB: «） |
| `»` | **すべて開く** | 右向き二重シェブロン（U+00BB: »） |

### 配置イメージ

```jsx
   «  »　[ − ]  100%  [ + ]
　 ↑  ↑              
閉じる 開く
```

### デザイン仕様

- ボタンの装飾は％表示の白い枠と同様のデザインの枠の中にテキストの `«` `»` を表示する
- ズームコントロールとの間に **16px程度のギャップ** を空ける
- `«` と `»` の間のギャップは **8px** 程度
- フォントサイズ：ズームコントロールと同じ（現行のUI基準に合わせる）
- 色：**#37352f**（Notionダークグレー）= 他のコントロールと統一
- ホバー時：**opacity: 0.6** 程度のフィードバック
- カーソル：**pointer**
- ツールチップ（title属性）：
    - `«` → `すべて閉じる`
    - `»` → `すべて開く`

### 動作

- `»`（すべて開く）クリック → マインドマップの **全折りたたみノードを一括展開**（現行の「すべて開く」ボタンと同じ動作）
- `«`（すべて閉じる）クリック → マインドマップの **全ノードを一括折りたたみ**（ルートノードの直下の子ノードだけが見える状態にする）
- ツリーナビとの連動は現行通り（折りたたみ/展開がツリーナビにも反映）

# 5. 作業の進め方

1. まず既存のソースコードを確認し、変更が必要なファイルを特定する
2. 修正Aと修正Bは独立した変更なので、どちらから着手してもよい
3. 変更後、ビルド（`node build.js`）してから動作確認する
4. 完了後、`git add` → `git commit` → `git push` する

# 6. 注意事項

- 既存のマインドマップ操作（ノード追加・削除・移動・コピー等）に影響を与えないこと
- localStorageの既存データとの後方互換性を維持すること
- UIデザインはNotion風のクリーンなスタイルを維持すること（カラーパレットは現行仕様書の「10. デザイン仕様」に準拠）
- ブラウザ対応範囲は現行通り（Chrome, Edge, Safari, Firefox最新版）
