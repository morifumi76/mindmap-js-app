# 3. 修正C：Cmd/Ctrl + ドラッグでノード複製

## 3.1 現状

ノードをドラッグすると、そのノード（および子孫）が **移動** する。

複製（コピー）する機能は存在しない。

## 3.2 変更内容

### 基本動作

- ノードをドラッグ中に **`Cmd`（Mac）/ `Ctrl`（Windows）** を押すと、ドロップ時に **移動ではなく複製** になる
- Excel / PowerPoint の Cmd+ドラッグと同じ挙動
- 複製対象は、ドラッグ中のノードとその **子孫ノードすべて**
- ドロップ先は通常の移動と同じルールで決定する（ドロップ位置の親ノードの子として挿入）

### キー操作の詳細

- **ドラッグ中に Cmd/Ctrl を押す** → 複製モードに切り替わる（＋マーク表示）
- **ドラッグ中に Cmd/Ctrl を離す** → 通常の移動モードに戻る（＋マーク非表示）
- **ドラッグ開始前から Cmd/Ctrl を押している** → ドラッグ開始時点から複製モード
- **ドロップ時に Cmd/Ctrl が押されている** → 複製を実行
- **ドロップ時に Cmd/Ctrl が押されていない** → 通常の移動を実行

### 複製時の挙動

- ドラッグ元のノード（および子孫）は **そのまま残る**（元の位置に残存）
- ドロップ先に **新しいノード（深いコピー）** が作成される
- 複製されたノードには新しいIDが採番される
- 折りたたみ状態は複製元のまま引き継ぐ
- 複製後、複製先のノードが選択状態になる

### 複数選択 + 複製

- 複数ノードを選択した状態でドラッグ → Cmd/Ctrl を押すと、**選択中の全ノード** が複製対象になる
- 各ノードの子孫も含めて丸ごと複製

## 3.3 ＋マーク（複製インジケーター）の表示

ドラッグ中に Cmd/Ctrl が押されているとき、複製モードであることを視覚的に示す **＋マーク** を表示する。

### 単体ノードのドラッグ時

- ドラッグ中のノードの **右上** に小さい **＋マーク** を表示する
- 現行の複数選択時に赤丸で選択数が表示される場所と **同じ位置**

```
┌──────────────┐
│  ノード名     ⁺│  ← 右上に小さい＋マーク
└──────────────┘
```

### 複数選択 + 複製時

- 複数選択時は、赤丸の中に **選択数（例：③）** が表示される（現行機能）
- Cmd/Ctrl を押して複製モードになったとき、赤丸の数字の **右上に小さく＋マーク** を表示する
- 数字と＋マークが重ならないように、＋マークは数字の右上にオフセットして配置

```
複数選択のみ（Cmd/Ctrlなし）：     複数選択 + 複製（Cmd/Ctrl押下中）：
┌──────────────┐               ┌──────────────┐
│  ノード名     ③│               │  ノード名    ③⁺│  ← 赤丸数字の右上に＋
└──────────────┘               └──────────────┘
```

### ＋マークのデザイン仕様

| 項目 | 値 |
| --- | --- |
| 文字 | `+`（プラス記号） |
| フォントサイズ | 10px（赤丸の数字より小さめ） |
| 色 | #37352f（Notionダークグレー）または赤丸と同系色 |
| 背景 | 白の小さい丸（視認性確保）、またはドロップシャドウ付き |
| 位置（単体時） | 赤丸バッジと同じ位置（ノード右上） |
| 位置（複数選択時） | 赤丸バッジの右上にオフセット（被らないように） |
| 表示条件 | ドラッグ中 かつ Cmd/Ctrl が押されているときのみ |
| 非表示条件 | Cmd/Ctrl を離したとき、またはドロップ完了時 |

## 3.4 Undo/Redo対応

- 複製操作は `Cmd + Z` で **Undo可能** にする
- Undoすると、複製で作成されたノードが削除される（元のノードには影響なし）
- `Cmd + Y` で Redo可能

# 4. 作業の進め方

1. まず既存のソースコード（特に `drag.js`）を確認し、変更が必要なファイルを特定する
2. ドラッグ中の Cmd/Ctrl キー検出と＋マーク表示を実装する
3. ドロップ時の分岐（移動 or 複製）を実装する
4. 複数選択時の＋マーク位置調整を実装する
5. Undo/Redo 対応を実装する
6. 変更後、ビルド（`node build.js`）してから動作確認する
7. 完了後、`git add` → `git commit` → `git push` する

---

# 5. 注意事項

- 既存のドラッグ&ドロップ（移動）機能を壊さないこと
- Cmd/Ctrl を押していない場合の挙動は現行通り（移動）
- localStorageへの保存は、複製で新しいノードが追加された時点で自動保存（現行の保存タイミングと同じ）
- ブラウザ対応範囲は現行通り（Chrome, Edge, Safari, Firefox最新版）
- Mac は `Cmd`（metaKey）、Windows は `Ctrl`（ctrlKey）で判定すること
