# ズームコントロールバーの実装要件

## 1. 概要

マインドマップのノード数が増加した際に、手動でのサイズ調整が煩雑になる問題を解決するため、**ズーム率を操作するコントロールバー（UIコンポーネント）**を新設する。

---

## 2. UIレイアウト

### 配置場所

- 既存の **カラー切替ボタン（グレー●・イエロー●）の横（左隣）** に配置する
- ヘッダーバー内に収まるようにする

### コンポーネント構成（左から右の順）

```
[ − ]  100%  [ + ]
```

| 順番 | 要素 | 説明 |
| --- | --- | --- |
| 1 | **マイナスボタン**（ − ） | クリックするとズーム率が **5%ずつ減少** する |
| 2 | **ズーム率表示** | 現在のズーム率を `〇〇%` の形式で中央に表示する（初期値：`100%`） |
| 3 | **プラスボタン**（ + ） | クリックするとズーム率が **5%ずつ増加** する |

### デザインイメージ

```
[ − ]  100%  [ + ]  [カラー切替: ● ●]
```

---

## 3. 機能仕様

### ズーム操作

| 項目 | 仕様 |
| --- | --- |
| **刻み幅** | 5% 単位 |
| **初期値** | 100% |
| **最小値** | 10%（※調整可。これ以上縮小しても実用性がないため） |
| **最大値** | 200%（※調整可） |
| **操作方法** | **[ − ]** ボタンをクリック → 5%縮小
**[ + ]** ボタンをクリック → 5%拡大 |
| **表示更新** | ボタンクリックのたびに中央のパーセンテージ表示をリアルタイムで更新する |
| **ズーム対象** | マインドマップのキャンバス全体（全ノード・接続線を含む）を `transform: scale()` 等でスケーリングする |
| **ズームの基準点** | キャンバスの中心を基準にスケーリングする |

### 境界値の振る舞い

- 最小値（10%）に到達した場合 → **[ − ] ボタンを無効化**（disabled / グレーアウト）
- 最大値（200%）に到達した場合 → **[ + ] ボタンを無効化**（disabled / グレーアウト）
- 範囲内に戻ったら再度有効化する

---

## 4. スタイル要件

| 項目 | 仕様 |
| --- | --- |
| **全体のトーン** | 既存UIと統一感のあるミニマルなデザイン |
| **ボタンスタイル** | シンプルなアイコンボタン（`−` と `+`）。ホバー時にカーソルが pointer に変わる |
| **パーセンテージ表示** | 中央配置、フォントサイズは既存のヘッダーテキストと合わせる |
| **ホバーエフェクト** | ボタンにホバー時の視覚フィードバック（背景色の変化等）を付ける |
| **disabled時** | ボタンの色を薄くし、クリック不可にする（`pointer-events: none` or `opacity` 調整） |

---

## 5. 実装上の注意事項

- ズーム変更時、**ノードの位置やドラッグ操作のオフセット計算にスケール値を反映する**こと（ドラッグ位置がずれないようにする）
- ノードのクリック・編集等の**イベントハンドリングがスケール後も正常に動作する**ことを確認する
- パフォーマンスに配慮し、**スケーリングはCSS transform を優先的に使用**する（再描画を最小限に）
- 現在のズーム率は**状態変数として管理**し、他の機能（ノード追加等）からも参照可能にする

---

## 6. 将来の拡張候補（今回のスコープ外）

> 以下は今回の実装には含めないが、将来的に検討可能な機能
> 
- ズーム率の数値を直接クリックして手入力で変更
- キーボードショートカット（Ctrl + / Ctrl −）でのズーム操作
- マウスホイール（Ctrl + スクロール）でのズーム操作
- 「フィット」ボタン（全ノードが画面に収まるよう自動調整）
